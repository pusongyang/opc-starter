# AI 亲和度检查清单

本文档提供完整的检查清单和评分细则，适用于前端、后端、全栈等各类项目。

## 评分等级说明

| 等级 | 分数范围 | 说明 |
|------|----------|------|
| A | 90-100 | 优秀，AI 可高效协作 |
| B | 75-89 | 良好，大部分场景适用 |
| C | 60-74 | 及格，需要改进 |
| D | 40-59 | 较差，需系统性改造 |
| F | <40 | 极低，不适合 AI 协作 |

---

## 维度 1: 最小可运行环境（权重 12%）

**核心目标**: AI Coding 工具可在本地快速启动应用，验证修改效果

### 检查项

- [ ] **一键启动**
  - [ ] 有明确的启动命令（`npm run dev` / `make run` / `mvn spring-boot:run` / `go run cmd/...`）
  - [ ] 启动命令在 README 中有说明
  - [ ] 无需复杂前置配置即可运行

- [ ] **依赖屏蔽/Mock**
  - [ ] 前端：支持 Mock 模式（MSW / json-server / 环境变量开关）
  - [ ] 后端：支持本地数据库替代（H2 / SQLite / Testcontainers）
  - [ ] 后端：支持通过 Profile/Build Tag 屏蔽外部依赖
  - [ ] Mock 模式下核心功能可正常使用

- [ ] **环境变量管理**
  - [ ] 提供环境模板（`.env.example` / `application-local.properties` / `config/local.yaml`）
  - [ ] 文档说明必需和可选的配置项
  - [ ] 本地开发无需真实外部服务凭证

- [ ] **依赖锁定**
  - [ ] 使用锁文件（`package-lock.json` / `go.sum` / `Cargo.lock` / `poetry.lock`）
  - [ ] 可选：使用版本管理器（`.nvmrc` / `.python-version` / `.sdkmanrc`）
  - [ ] 可选：有构建包装器（`mvnw` / `gradlew`）

### 评分标准

| 分数 | 条件 |
|------|------|
| 4 | 一键启动 + 依赖可屏蔽 + 环境模板 + 依赖锁定 |
| 3 | 一键启动 + 有启动文档或部分 Mock + 依赖锁定 |
| 2 | 可启动但需要手动配置环境变量或部分依赖 |
| 1 | 启动依赖外部服务，无 Mock 方案 |
| 0 | 无明确启动方式或强依赖外部服务无法本地运行 |

## 维度 2: 类型系统与静态分析（权重 12%）

**核心目标**: 静态类型检查提供快速反馈，AI 生成的代码可被即时校验

### 检查项

- [ ] **类型系统**
  - [ ] TypeScript：`tsconfig.json` 配置合理
  - [ ] Go：内置强类型 + `go vet`
  - [ ] Java/Kotlin：编译时类型检查
  - [ ] Python：mypy / pyright 配置
  - [ ] Rust：内置强类型

- [ ] **严格模式**
  - [ ] TypeScript：`strict: true`，无 `any` 滥用
  - [ ] Go：`go vet` + staticcheck / golangci-lint
  - [ ] Java：Error-Prone / NullAway / SpotBugs
  - [ ] Python：`mypy --strict`

- [ ] **类型覆盖**
  - [ ] 函数参数和返回值有类型
  - [ ] 外部依赖有类型定义
  - [ ] 无类型逃逸（any / unsafe / suppress）

### 评分标准

| 分数 | 条件 |
|------|------|
| 4 | 强类型 + 严格模式 + 静态分析工具 + 无类型逃逸 |
| 3 | 有类型系统，少量类型逃逸 |
| 2 | 有类型系统但配置宽松 |
| 1 | 部分代码有类型 |
| 0 | 动态语言无类型注解 |

---

## 维度 3: 测试体系（权重 15%）

**核心目标**: AI 修改代码后可快速运行测试验证正确性

### 检查项

- [ ] **测试框架**
  - [ ] 配置测试框架（Jest/Vitest / Go testing / JUnit/TestNG / pytest）
  - [ ] 测试可一键运行（`npm test` / `go test ./...` / `mvn test` / `pytest`）
  - [ ] 测试运行速度合理（单元测试 < 2分钟）

- [ ] **覆盖率**
  - [ ] 配置覆盖率报告（Istanbul / JaCoCo / coverage.py / Go cover）
  - [ ] 总体覆盖率 > 50%
  - [ ] 核心模块覆盖率 > 70%

- [ ] **测试可独立运行**
  - [ ] 不依赖外部数据库（使用 H2/SQLite/内存数据库/Testcontainers）
  - [ ] 不依赖外部 API（使用 mock/fixture/stub）
  - [ ] 测试间无状态依赖

- [ ] **E2E/集成测试**（可选加分）
  - [ ] 前端：Cypress / Playwright 覆盖关键流程
  - [ ] 后端：集成测试覆盖核心 API
  - [ ] 契约测试验证接口兼容性

### 评分标准

| 分数 | 条件 |
|------|------|
| 4 | 覆盖率 >70% + 核心路径有测试 + 测试可独立运行 + E2E/集成测试 |
| 3 | 覆盖率 50-70% + 有单元测试框架 + 部分 mock |
| 2 | 有测试但覆盖率低或依赖外部服务 |
| 1 | 有测试框架但测试极少 |
| 0 | 无测试 |

```

= (D1/4 × 12) + (D2/4 × 12) + (D3/4 × 15) + (D4/4 × 12) + (D5/4 × 8)
  + (D6/4 × 10) + (D7/4 × 10) + (D8/4 × 8) + (D9/4 × 8) + (D10/4 × 5)
```

权重总和 = 100%
